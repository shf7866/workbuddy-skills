# 职业生涯研究 · 理论选择推荐工作流

> 输入：研究问题 / 现象描述 / 选题方向
> 输出：适配理论 + 研究设计建议 + 可操作化假设命题

---

## 使用方式

**标准流程**：用户描述研究问题 → 执行五步工作流 → 输出理论推荐+研究设计

**快速模式**：用户直接说理论名称 → 跳至 Step 5 输出6维度速查

**Specky模式**：用户说"Specky要做XX功能" → 执行 Step 1-2 识别理论 → 跳至 Step 5 产品化建议

## 工作流

### Step 1：问题结构化

**输入**：用户的研究问题或现象描述

**操作**：
1. 提取核心机制关键词（匹配/发展/决策/动机/系统/构建）
2. 识别研究层次：个体层 / 组织层 / 社会层
3. 识别时间取向：横截面 / 纵向 / 生涯追踪

**输出**：结构化标签 `机制类型 + 研究层次 + 时间取向`

**失败模式**（if-then 格式）：
- IF 用户输入过于模糊（如"我想研究职业生涯"）
  THEN 输出结构化追问模板（5个具体问题），要求用户补充
  ELSE IF 用户补充后仍模糊
    THEN 给出3个最常见研究问题示例供选择
- IF 用户输入非职业生涯领域
  THEN 明确告知本skill覆盖41个职业生涯理论，建议换用其他理论检索工具

**🔴 CHECKPOINT**：结构化标签生成后，向用户确认"我理解您的研究问题是：XXX，对吗？"获得确认再进入Step 2。

---

### Step 2：理论候选匹配

**输入**：Step 1 的结构化标签

**操作**：
1. 查「理论选择决策树」（见附录A）匹配候选理论
2. 【语义匹配】如果决策树无法精确匹配，根据理论核心命题与用户问题的语义相似度，推荐最相关的2-3个理论
   - 计算方法：比对用户问题中的关键词与理论核心命题中的关键词（见附录C或对应文件）
   - 优先推荐核心命题可直接操作化为假设命题的理论
3. 至少推荐2个理论（主理论+备选理论）
3. **读取对应文件获取完整6维度内容**（按以下映射，搜索 `## [理论名]` 章节）：
> **具体读取指引**：
> - 读取文件后，搜索 `## [理论名]` 章节（如 `## 1. 人—职匹配理论`）
> - 提取该章节下的6个二级标题：核心概念/研究方法适配/实践应用/当代青年指引/理论局限与争议/关键文献
> - 如果文件内容过长（>1000字），先输出核心概念+研究方法适配，询问用户是否需要展开其他部分
> 
> **文件与理论映射**（精确搜索关键词）：
> - 理论#1-#5：搜索 `## 1.` / `## 2.` / `## 3.` / `## 4.` / `## 5.`
> - 理论#6-#12：搜索 `## 6.` / `## 7.` / ... / `## 12.`
> - 理论#13-#17：搜索 `## 13.` / ... / `## 17.`
> - 理论#18-#28：搜索 `## 18.` / ... / `## 28.`
> - 理论#29-#34：搜索 `## 29.` / ... / `## 34.`
> - 理论#35-#41：搜索 `## 35.` / ... / `## 41.`
> 
> 如果搜索不到确切章节，读取文件开头部分（前200行），根据章节标题手动定位。
5. 每个理论输出：匹配理由/核心命题/研究方法适配摘要
**输出格式**：
```
### 推荐理论
1. **[理论名]**（匹配度：高/中/低）
   - 匹配理由：[具体理由]
   - 核心命题：[可操作化的命题，可直接写成H1]
   - 研究方法适配：[设计类型]；测量工具：[具体量表]
   
2. **[备选理论名]**（匹配度：中/低）
   - 匹配理由：...
   - 核心命题：...
   - 研究方法适配：...
```
**决策树规则**：
- 涉及人岗/人组织匹配 → 匹配理论簇（#1, #3, #5）
- 涉及生涯发展阶段/身份建构 → 发展理论簇（#6, #7, #8, #9）
- 涉及职业决策困难/决策过程 → 决策理论簇（#13, #15, #16）
- 涉及动机/自我效能/呼唤 → 动机理论簇（#19, #21, #26）
- 涉及工作家庭冲突/系统互动 → 系统理论簇（#29, #30, #34）
- 涉及无边界/主动建构/意义 → 构建理论簇（#35, #37, #38, #41）

**输出**：候选理论列表（含匹配理由）

**失败模式**（if-then 格式）：
- IF 决策树无法匹配（用户输入跨簇问题）
  THEN 返回所有相关簇的理论，要求用户选择侧重方向
  ELSE IF 用户仍无法选择
    THEN 建议用户先阅读附录C速查表，再回来咨询
- IF 用户问题涉及非职业生涯领域理论
  THEN 明确告知边界，建议咨询对应领域专家

### Step 3：研究方法适配

**输入**：Step 2 的候选理论

**操作**：
对每个候选理论：
1. **读取对应文件**（映射见Step 2）中的"研究方法适配"章节
2. 提取以下信息，填入「研究方法适配卡」：
   - **推荐研究设计**：量化/质性/混合；横截面/纵向/ESM/实验
   - **典型测量工具**：量表名称 + 作者 + 年份；中文版开发情况（已开发/需自行翻译）
   - **变量设计建议**：IV/DV/Mediator/Modifier 各推荐2-3个具体变量名
   - **样本要求**：量化设计的最小样本量（根据推荐统计方法）；质性设计的建议样本量（n=？）
   - **数据分析方法**：相关/回归/SEM/多层模型/主题分析/...

**输出格式**（每个候选理论一张卡）：
```
#### [理论名] 研究方法适配卡

| 维度 | 具体内容 |
|------|---------|
| 推荐研究设计 | [量化/质性/混合]；[横截面/纵向/ESM/实验] |
| 典型测量工具 | [量表名]（[作者]，[年份]）<br>中文版：[已开发/需自行翻译] |
| IV建议 | 1. [具体变量名] 2. [具体变量名] |
| DV建议 | 1. [具体变量名] 2. [具体变量名] |
| Mediator建议 | 1. [具体变量名] 2. [具体变量名] |
| Moderator建议 | 1. [具体变量名（调节变量）] 2. [具体变量名] |
| 样本要求 | 量化：最小N=[具体数字]；质性：建议n=[具体数字] |
| 数据分析方法 | [相关/回归/SEM/多层模型/主题分析/...] |
```
**输出**：每个候选理论的「研究方法适配卡」

**失败模式**（if-then 格式）：
- IF 理论X的测量工具在中国语境下未开发
  THEN 明确告知，推荐相近替代工具或建议用户自行开发
  ELSE IF 用户坚持使用该理论
    THEN 提供英文原版量表 + 翻译指南
- IF 用户要求"最省时间"的设计
  THEN 优先推荐横截面+现成量表，同时注明局限性

### Step 4：假设命题生成

**输入**：Step 2 的候选理论 + Step 3 的研究方法

**操作**：
1. 从Step 2获取的"核心命题"中提取可直接操作化的假设命题
2. 每个假设命题格式（严格）：
   - `H[数字]: [IV] 正向预测 [DV]` （如理论预测负向，用"负向预测"）
   - `H[数字]: [Mediator] 中介 [IV] 与 [DV] 的关系`
   - `H[数字]: [Modifier] 调节 [IV] 与 [DV] 的关系`
3. 标注每个命题的理论来源（学者 + 年份 + 文献）
4. 如用户要求"创新假设"，触发附录B（理论跨界组合）

**输出格式**：
```
### 假设命题组（基于[理论名]）

#### 主理论：[理论名]
- H1: [IV] 正向预测 [DV]（理论来源：[学者]，[年份]）
- H2: [Mediator] 中介 [IV] 与 [DV] 的关系（理论来源：[学者]，[年份]）
- H3: [Modifier] 调节 [IV] 与 [DV] 的关系——当[条件]时，[IV]对[DV]的影响更强（理论来源：[学者]，[年份]）

#### 备选理论：[理论名]
- H1: ...
- H2: ...
```
**输出**：假设命题组（每个候选理论2-3个命题）

**失败模式**（if-then 格式）：
- IF 理论与现象语义相关但逻辑链不清晰
  THEN 标注"需补充逻辑链"，给出2个可能的桥梁构念
  ELSE IF 用户拒绝提供的桥梁构念
    THEN 建议用户先做探索性访谈（n=5-8）
- IF 用户要求"创新假设"
  THEN 触发理论跨界组合（见附录B），生成跨理论假设

**🔴 CHECKPOINT**：假设命题生成后，询问用户"这些假设是否符合您的研究意图？需要增加/删除/修改哪个？"

```
### 推荐理论（按适配度排序）
1. [理论名]（匹配度：高/中/低）

   - 核心命题：[可直接操作化的命题]
   
### 研究设计建议
| 理论 | 推荐设计 | 测量工具 | 最小样本量 |
|------|---------|---------|------------|

### 假设命题
- H1: ...
- H2: ...

### 当代青年指引（如适用）
[该理论对当代中国青年现象的解释力]

### 理论局限与边界
[使用该理论时需注意的边界条件]

### 关键文献（供深入阅读）
- 奠基文献：...
- 近5年进展：...
```

**Specky产品模式**输出格式（当用户说"Specky要做XX功能"时）：
```
## Specky功能理论支撑报告

### 核心支撑理论
[理论名]：核心逻辑如何支撑该功能

### 产品化建议
- 测评模块设计（如有量表）
- 干预流程设计（如有咨询/辅导场景）
- 数据埋点建议（如需追踪行为）

### 竞品对标
[现有产品是否用了这个理论]
```

**失败模式**（if-then 格式）：
- IF 输出内容过长（>2000字）
  THEN 分段输出，先给「推荐理论+研究设计」部分，询问是否需要展开假设命题部分
  ELSE IF 用户确认需要展开
    THEN 继续输出假设命题部分
- IF 用户要求"只要理论名称，不要这么多内容"
  THEN 切换快速模式，只输出理论名称列表
  ELSE IF 用户要求"生成PDF"
    THEN 告知本skill不支持直接生成PDF，但可提供Markdown格式供用户自行转换
